# 日報 2026-05-12 — 第5弾の2: チームコラボ機能 (招待リンク + 共同閲覧)

## 実装サマリー

localStorage ベースのシングルユーザー運用に「チームワークスペース」レイヤーを追加。Pro/Studio プランで謳っているチーム共有機能の UI を完成させた。

---

## 追加ファイル

### `src/types/team.ts`
- `TeamMember` — id / email / displayName / role (`owner|admin|editor|viewer`) / joinedAt
- `Workspace` — id / name / ownerEmail / members / invites / createdAt
- `Invite` — code / email? / role / expiresAt / createdAt / usedAt?
- `ShareRecord` — resourceType / resourceId / sharedWith

### `src/hooks/useWorkspace.ts`
- localStorage `core_workspace_v1` で 1 ワークスペースを永続化
- `createWorkspace(name)` — ログインユーザーをオーナーとして初期化
- `generateInvite(role, expiresInDays=7)` — ランダム 10 文字コードを生成
- `redeemInvite(code)` — 期限・使用済みチェック → メンバー追加
- `removeMember(id)` / `changeRole(id, role)` / `revokeInvite(code)`
- 他タブとの `storage` イベント同期

### `src/components/TeamHub.tsx`
- ワークスペース作成フォーム (未作成時)
- メンバー一覧 (アバター + ロール pill + ロール変更 + 削除)
- メンバー上限チェック → 超過時アップグレード促進 modal
- 「招待リンクを生成」— ロール選択 → コード生成 → URL コピー (`https://core-prism-app.vercel.app/?invite=CODE`)
- 有効招待一覧 (有効期限残り日数 / 無効化ボタン)
- データ共有スナップショット (デモ) — オーナーが localStorage を保存 → メンバーが同期して取り込む

### `src/components/AcceptInviteModal.tsx`
- URL `?invite=CODE` を sessionStorage 経由で受け取りモーダル表示
- 「参加する」→ `redeemInvite(code)` → 成功でモーダル閉じ
- エラー (期限切れ / 使用済み) はインライン表示
