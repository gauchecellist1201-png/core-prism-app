### 実装詳細

#### `api/stripe/checkout.ts`
- POST `{ plan, brand, email? }` → Stripe Checkout Session を生成
- Price ID は env (`STRIPE_PRICE_LITE` 等) でブランド × プランごとに指定
- `success_url = ${origin}/billing/success?session_id={CHECKOUT_SESSION_ID}&brand=${brand}`
- `cancel_url` はブランド別 LP へ
- env 未設定時: `{ error: "STRIPE_NOT_CONFIGURED" }` + HTTP 503
- Stripe SDK 非依存 (直接 fetch + `application/x-www-form-urlencoded`)

#### `api/stripe/webhook.ts`
- Stripe-Signature ヘッダを Web Crypto (HMAC-SHA256) で手動検証
- タイムスタンプ 5 分超のリクエストは拒否
- KV 永続化なし: 検証 + 200 のみ (success_url の session_id でクライアントが取得)
- `STRIPE_WEBHOOK_SECRET` 未設定時は 200 + warning を返す (開発フレンドリー)

#### `api/billing/lookup.ts`
- GET `?session_id=cs_xxx` → Stripe セッションを `expand[]=subscription` で取得
- Response: `{ plan, brand, status, customer_email, subscription_id, current_period_end }`

#### `api/stripe/cancel.ts`
- POST `{ subscription_id }` → `cancel_at_period_end=true` で即解約せず期末解約
- Response: `{ success: true, cancel_at }`

#### `api/email/send.ts`
- POST `{ to, template, data }` → Resend API でメール送信
- テンプレート 3 種 (日本語 HTML):
  - `welcome`: 登録完了 + 3 ステップ始め方ガイド
  - `trial_ending`: トライアル残 N 日 + アップグレード CTA
  - `cancel_save`: 解約完了 + 復帰クーポン `COMEBACK50` (50% OFF 30 日有効)
- from: `EMAIL_FROM` env または `noreply@coreprism.app`

#### `src/lib/billing.ts` 変更点
- `BillingUser` インタフェースに `subscriptionId?: string` と `currentPeriodEnd?: number` を追加
- `syncFromStripe(sessionId)` 関数を追加:
  - `/api/billing/lookup` を呼び、成功すれば localStorage の user を更新
  - マスターモード (`GAUCHE2026`) はバイパス

#### `src/components/CheckoutModal.tsx` 変更点
- 決済実行時に `/api/stripe/checkout` を呼ぶ
- 503 (STRIPE_NOT_CONFIGURED) → テストモードに自動フォールバック
- 本番モード: signup → welcome メール送信 (非同期) → Stripe Checkout へリダイレクト
- テストモード: signup → welcome メール → success ステップ表示

#### `src/components/BillingSuccess.tsx` (新規)
- `/billing/success?session_id=...&brand=...` でマウント
- `syncFromStripe()` でプラン確定 → 2 秒後にアプリへリダイレクト
- エラー時: アプリへ進むリンク + サポート問い合わせリンク

#### `src/components/BillingDashboard.tsx` (新規)
- 現在のプラン・料金・次回更新日を表示
- 解約フロー: 確認ダイアログ → POST `/api/stripe/cancel` → `cancel_save` メール送信
