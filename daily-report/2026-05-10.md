# 日報 2026-05-10 — 第4弾の2: Stripe 本番接続 + メール通知 (Resend)

## 実装概要

Stripe Checkout セッション発行・Webhook 受信・サブスク照会・解約 API と
Resend によるメール通知を追加し、CheckoutModal を本番 Stripe に対応させた。
env 未設定でも ¥0 確認用モードにフォールバックするため既存ユーザーへの影響なし。

## フロー説明

```
[CheckoutModal] 決済ボタン
  ↓ POST /api/stripe/checkout { plan, brand, email }
  ↓ (503: env未設定) → ¥0 テストモード (既存フロー維持)
  ↓ (200: url) → window.location.href = stripe_checkout_url
  ↓ Stripe 決済完了
  ↓ success_url = /billing/success?session_id={CHECKOUT_SESSION_ID}&brand=...
  ↓ [BillingSuccess] syncFromStripe(sessionId)
  ↓   → GET /api/billing/lookup?session_id=...
  ↓   → localStorage の BillingUser.plan を更新
  ↓ 3秒後アプリへリダイレクト
```

```
[BillingDashboard] 解約ボタン
  ↓ POST /api/stripe/cancel { subscription_id }
  ↓ cancel_at_period_end: true
  ↓ sendEmail(email, 'cancel_save', { couponCode: 'COMEBACK50' })
```

## 必要な環境変数 (Vercel Dashboard に設定)

```env
# Stripe
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...

# Iris プラン Price ID
STRIPE_PRICE_LITE=price_...
STRIPE_PRICE_STANDARD=price_...
STRIPE_PRICE_PRO=price_...
STRIPE_PRICE_STUDIO=price_...

# Prism プラン Price ID
STRIPE_PRICE_PRISM_STARTER=price_...
STRIPE_PRICE_PRISM_STANDARD=price_...
STRIPE_PRICE_PRISM_EXCLUSIVE=price_...

# Resend
RESEND_API_KEY=re_...
EMAIL_FROM=noreply@coreprism.app
```

> **未設定時の挙動**: STRIPE_SECRET_KEY 未設定 → checkout が 503 → CheckoutModal が ¥0 テストモードにフォールバック。RESEND_API_KEY 未設定 → send が 503 → emailNotify がコンソールログのみ。アプリは正常動作継続。

## Stripe Webhook 設定手順

1. Stripe Dashboard → Developers → Webhooks → `+ Add endpoint`
2. URL: `https://core-prism-app.vercel.app/api/stripe/webhook`
3. Events: `checkout.session.completed`, `customer.subscription.created`, `customer.subscription.updated`, `customer.subscription.deleted`
4. 発行された `whsec_...` を Vercel の `STRIPE_WEBHOOK_SECRET` に設定
