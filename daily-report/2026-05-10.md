## 実装の設計方針

### Stripe Checkout フロー
1. ユーザーが CheckoutModal でプラン選択・アカウント作成
2. `/api/stripe/checkout` を叩き Checkout Session を作成
3. `session.url` へリダイレクト (Stripe ホステッド決済ページ)
4. 決済完了後 `/billing/success?session_id=cs_xxx&brand=prism` へ戻る
5. `syncFromStripe(sessionId)` で `/api/billing/lookup` を叩きプランを localStorage に同期
6. 2 秒後にダッシュボードへ自動リダイレクト

### Webhook
- 署名検証のみ (200 を返す)
- KV/DB なしのシンプル設計
- 将来: `checkout.session.completed` でサーバー側プラン更新を追加可能

### env 未設定時の挙動
| 条件 | 挙動 |
|---|---|
| `STRIPE_SECRET_KEY` 未設定 | 503 + `STRIPE_NOT_CONFIGURED` → CheckoutModal が test mode (¥0) で続行 |
| `STRIPE_PRICE_*` 未設定 | 503 + `PRICE_NOT_CONFIGURED` → 同上 |
| `RESEND_API_KEY` 未設定 | 503 + `EMAIL_NOT_CONFIGURED` → UI には影響なし (fire and forget) |

## 必要な環境変数 (Vercel ダッシュボードに設定)

```
# Stripe
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...

# Iris プラン Price ID
STRIPE_PRICE_LITE=price_...
STRIPE_PRICE_STANDARD=price_...
STRIPE_PRICE_PRO=price_...
STRIPE_PRICE_STUDIO=price_...

# Prism プラン Price ID
STRIPE_PRICE_PRISM_STARTER=price_...
STRIPE_PRICE_PRISM_STANDARD=price_...
STRIPE_PRICE_PRISM_EXCLUSIVE=price_...

# Resend
RESEND_API_KEY=re_...
EMAIL_FROM=noreply@coreprism.app
```

### Stripe Webhook エンドポイント設定
- URL: `https://core-prism-app.vercel.app/api/stripe/webhook`
- イベント: `checkout.session.completed`, `customer.subscription.created`, `customer.subscription.updated`, `customer.subscription.deleted`
